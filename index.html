História da Web

A história da Internet está diretamente ligada à evolução da tecnologia como um todo. Desde o surgimento dos primeiros computadores, como o Harvard Mark I, o Colossus e o ENIAC, que eram máquinas enormes e funcionavam com válvulas, até os computadores atuais, ultrafinos e portáteis, equipados com uma quantidade enorme de chips de circuitos integrados.

A Internet surgiu inicialmente para possibilitar a comunicação entre universidades, e posteriormente foi utilizada no ambiente militar. Logo, percebeu-se que ela poderia ser amplamente utilizada e foi difundida em todas as áreas de atuação.

O que são Clients?

Cliente é o termo utilizado para se referir ao lado do usuário em uma aplicação ou programa, que recebe os dados do servidor para utilizá-los em sua máquina local.

Navegadores são programas usados para a interpretação das páginas HTML recebidas do servidor. Há uma grande variedade de navegadores, também conhecidos como "browsers", porém a interpretação do arquivo pode variar de um para o outro dependendo da tecnologia utilizada.

Aplicações Web são soluções que utilizam a internet como meio de comunicação, não sendo necessária a instalação de qualquer programa específico para sua utilização.

Dispositivos móveis são equipamentos que podem se conectar à internet e serem usados de praticamente qualquer lugar, como, por exemplo, laptops, tablets, smartwatches e smartphones.

O que são Servidores?

Servidores são computadores com maior potência e desempenho, que são utilizados para receber solicitações ou executar aplicações requisitadas pelos Clients (usuários).

Existem diversos tipos de servidores, e alguns exemplos foram apresentados:

Servidor de Arquivos
É um servidor onde é possível armazenar arquivos, e um exemplo comum é o Google Drive.
Servidor Firewall (Segurança)
É um servidor utilizado para configurar as especificações de segurança de uma rede, determinando as possibilidades de entrada e saída de pacotes de dados na mesma.
Servidor de Streaming
É um servidor onde as aplicações de Streaming são executadas, e um exemplo conhecido é o Netflix.
Servidor de E-mail
É um servidor utilizado para armazenar os e-mails enviados e recebidos de um domínio ou rede específica. Exemplos populares incluem o Outlook, amplamente utilizado, e o Gmail da Google.
Servidor Web
É um servidor onde as páginas da Web são armazenadas, contendo todos os arquivos que fazem parte delas, como os arquivos HTML, CSS e JavaScript.

O que é HTML

O HTML, sigla em inglês para "Linguagem de Marcação de Hipertexto", foi criado pelo britânico Tim Berners-Lee. Essa linguagem é o componente básico da web e permite inserir conteúdo e estabelecer a estrutura básica de um site. Em outras palavras, o HTML serve para dar significado e organizar as informações em uma página da web. Sem ele, os navegadores não saberiam como exibir textos como elementos ou carregar imagens e outros conteúdos.

Como funciona o HTML?

Quando um arquivo HTML é carregado em um navegador, o navegador faz a leitura do arquivo e exibe o seu conteúdo para o usuário final. Esse arquivo pode ser visualizado em qualquer navegador, como Google Chrome, Edge ou Mozilla Firefox. Um site geralmente é composto por várias páginas HTML. Por exemplo, um site com três páginas (uma página inicial, uma página de contato e uma página de produtos) terá pelo menos três arquivos HTML distintos, um para cada página do site.

Como é uma estrutura básica de um HTML ?

O documento começa com uma declaração do tipo de documento, que, no caso do HTML5, pode ser declarado com a tag <!DOCTYPE html>. Em seguida, iniciamos nosso documento com a tag <html>, que deve ser fechada no final do documento com a tag </html>. Depois, o documento deve ter um elemento <head>, que receberá todas as informações básicas da página, como o título, links para elementos externos, metadados, etc. É aqui que definimos o título da página com a tag <title>. Finalmente, abrimos a tag <body>, que representa o corpo do nosso documento, onde ficarão todos os elementos que serão renderizados na tela do navegador. Desta forma, temos a estrutura básica do nosso documento HTML: 

<!DOCTYPE html>
        <html>
            <head>
                <title></title>
            </head>
        <body>
        
        </body>

Para evitar erros de renderização ou incompatibilidade de caracteres, também devemos declarar o padrão de codificação. No Brasil e em grande parte da América Latina, usamos o padrão UTF-8. Para fazer isso, usamos a tag de metadados <meta/> e declaramos no atributo charset que nosso padrão é o UTF-8. Assim, temos a estrutura básica do nosso documento HTML:

<!DOCTYPE html>
            <html>
                <head>
                    <title></title>
                    <meta charset="utf-8"/>
                </head>
